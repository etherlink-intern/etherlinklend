# Security Policy

Responsible disclosure email: `security@example.invalid` (placeholder; replace before launch).

No bug bounty is promised unless a formal program is configured and published.

Severity classification: critical, high, medium, low, informational. Scope is repository code and modifications only; unmodified upstream submodules should be reported upstream unless the issue arises from this integration.

Reporters are asked not to exploit live funds, disclose publicly before embargo resolution, access private data, or disrupt network operations. Embargoed reports will be triaged, acknowledged, reproduced, fixed, reviewed, and disclosed after safe remediation.
